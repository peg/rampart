A different approach

Your agent needs real access to do real work.
It just shouldn't have access to everything.

Sandboxes lock everything down. Rampart lets your agent run npm install, hit staging, and run tests. It blocks credential reads, exfiltration, and destructive commands.

Sandbox approach

Lock everything, allow a few things

Agent can't install deps, run tests, or hit your staging API. Secure, but you fight the sandbox to get work done.

$ sandbox --allow ./project -- agent
✗ npm install        # denied
✗ pytest --db-url ... # denied
✗ curl staging.api    # denied
Rampart approach

Allow everything, block what's dangerous

Agent works normally. Rampart only intervenes on credential access, exfiltration, and destructive commands.

$ rampart setup claude-code
✔ npm install        # allowed
✔ pytest --db-url ... # allowed
✗ cat ~/.ssh/id_rsa   # denied

How it works

A seatbelt, not a cage

One Go binary, no cloud, no latency your agent can feel. See how we map to the OWASP Agentic Top 10.

🔍

Generate rules from behavior

Don't write policy from scratch. Run your agent in monitor mode, then rampart init --from-audit generates rules from what it actually did. Review, tweak, enforce.

record → enforce
🧠

Response scanning

Most tools only check what goes in. Rampart also checks what comes out. If a file read returns an AWS key, the response gets blocked before your agent sees it.

⏳

Approval gates

Mark any rule require_approval. The agent stops, you get a ping on Discord or Slack, and nothing runs until you say so.

🛡️

Agents can't whitelist themselves

Prompt injection tells your agent to weaken its own policy? Rampart blocks CLI self-modification. The agent can't rampart allow its way out.

🔗

Hash-chained audit log

Every decision logged with a SHA-256 chain. Tamper with one record and the whole chain breaks. Ships syslog and CEF if your team runs a SIEM.

🔌

MCP proxy

rampart mcp scan reads an MCP server's tool list and generates policy automatically. Drop-in proxy for Claude Desktop and any MCP client.

Compatibility

Hooks into what you already use

Native hooks for the big ones. MCP proxy for Claude Desktop. Shell wrapper and HTTP API for everything else.
